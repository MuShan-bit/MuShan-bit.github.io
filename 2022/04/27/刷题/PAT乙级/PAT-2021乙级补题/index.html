# 7-1: 好数

# 暴力

#

# 题目大意

满足 a2+ab+b2a^2+ab+b^2a2+ab+b2​​ 那么 a 和 b 就是好数的源头，然后输入一些数字，判断是不是好数，如果是输出 yes 并输出源头，否则输出 No 找到比它大的最小好数，并输出其源头

# 解题思路

因为这个题目数据量不大，设置好区间就可以暴力了

a 从 1 开始，b 从 a + 1 开始，保证不重复求出a2+ab+b2a^2+ab+b^2a2+ab+b2 的值为一个好数，由此筛出所有需要的好数

并使用 vector 存储好数的两个源头，有其他源头的时候直接 push_back, 最后肯定是满足题目要求的递增顺序

然后就是开始判断每个 num 是否为好数，如果正确直接遍历输出 vector [num] 的内容

否则在循环遍历找到比它大的第一个好数 (存一下更好) 但是题目数据量小暴力就行

# 失分原因

未知，同样的思路第二遍写就过了

# 7-3 自定义判题程序

# 模拟题

在每次允许插入、删除、修改一个字符的前提下，用最少的动作把一个字符串变成另一个字符串，是一道著名的可以用动态规划解决的问题。但判题的麻烦之处在于，虽然最小代价是唯一的，但变换方法却是不唯一的。例如把 PAT 变成 PTA 最少需要 2 步，可以保持第 1 个字母不变，修改后面 2 个字母，也可以保持第 1、2 个字母不变，在 A 前面插入 T，后面删除 T。由于拼题 A 系统的默认判题程序只能通过比对输出文件来判断对错，对这种正确答案输出不唯一的题目就不能处理了，需要出题者额外编写一个自定义判题程序来解决问题。

本题就请你编写这个自定义判题程序，读入两个字符串和用户程序产生的输出结果，判断他们的程序输出是否正确。

# 解题思路

先根据输入的处理结果处理字符串，然后将新生成的字符串对比原字符串判断是否一致

0: 直接把原字符放里

1: 删除，什么多不做

2: 改变字符，放入一个不会存在题目中的字符 ^ * & 之类的，可以表示任意字符

3: 插入字符：这个位置是重点，我研究了好长时间，题目说的是前面或者后面插入，但是我看了大佬的 ac 代码

他只写了在前面插入，就 ac 了，这个题目真的屑！辣鸡！

# 零分原因

卡在第三个插入的那个点，想了一会改了半天没弄出来，没尝试骗分，直接跳题了。之后没有时间补了

# 7-4 数组与链表

# 模拟题

# 题目大意

前面输入一些数据，由此表示出一个由多段内存组成的一个数组

给出一些数组下标，判断这些位置是否能存入数据到数组中，如果不能输出 Illegal Access

否则返回出元素地址，元素地址公式: 4 * 数组分出来的那小段的下标 + 当前段的起始地址

# 解题思路
